How It Works
5-Layer AI Pipeline
🎙️
L1 · Azure Whisper
Transcription
Video → parallel audio chunks → full text transcript
📋
L2 · GPT-4 Turbo
BRD Draft
Transcript + docs → structured HTML BRD document
🔍
L3 · GPT-4 Turbo
Gap Analysis
BRD → targeted questions about missing requirements
💬
L4 · Bedrock Claude
Chatbot Q&A
Conversational refinement with LangChain memory
✨
L5 · GPT-4 Turbo
Final BRD
BRD + chat context → polished, downloadable document
REST API
Endpoints
POST /brdfrdgeneration/upload Upload PDF, DOCX, TXT, MP4, or MP3 — triggers L1 + L2 + L3 pipeline AI Pipeline
POST /brdfrdgeneration/chatbot Send/receive chatbot messages — L4 conversational refinement via Bedrock Claude AI · LangChain
POST /brdfrdgeneration/final Generate final BRD — L5 synthesis using all context gathered so far AI Pipeline
POST /brdfrdgeneration/downloadbrd Export BRD as a streamed .docx Word document (HTML → python-docx) Export
POST /brdfrdgeneration/uploadvialink Download OneDrive meeting recording via Microsoft Graph API, then process MS Graph
GET /brdfrdgeneration/listfiles List all session files for a userId (Firestore-backed) Session
POST /brdfrdgeneration/delete Delete session from Firestore + GCS storage Session
GET /brdfrdgeneration/authsuccesslogin SSO callback — decodes JWT, creates session token, redirects user Auth · JWT
Quick Start
Example Usage
# 1. Upload a meeting recording → triggers full L1+L2+L3 pipeline curl -X POST http://localhost:8501/brdfrdgeneration/upload \ -F "userId=analyst_01" \ -F "files=@quarterly_planning.mp4" # Response (L2 BRD + session ID) { "html": "<h1>Business Requirements Document</h1>...", "unique_id": "3f7a9b2c-1d4e-..." } # 2. Also works with PDF documents curl -X POST http://localhost:8501/brdfrdgeneration/upload \ -F "userId=analyst_01" \ -F "files=@project_brief.pdf" # 3. Or via OneDrive link (shared meeting recording) curl -X POST http://localhost:8501/brdfrdgeneration/uploadvialink \ -F "userId=analyst_01" \ -F "meeting_url=https://mahindra-my.sharepoint.com/..."
# Start chatbot session (L4 — AWS Bedrock Claude 3.5) curl -X POST http://localhost:8501/brdfrdgeneration/chatbot \ -H "Content-Type: application/json" \ -d '{ "userId": "analyst_01", "uniqueId": "3f7a9b2c-1d4e-...", "file_name": "quarterly_planning", "question": "hi" }' # Bot responds with a targeted gap-filling question: { "status": "success", "bot_response": "Who are the primary stakeholders for this system? The BRD doesn't clearly define the approval chain...", "session_id": "3f7a9b2c-1d4e-..." } # Continue conversation — bot uses LangChain memory (last 20 turns) curl -X POST http://localhost:8501/brdfrdgeneration/chatbot \ -H "Content-Type: application/json" \ -d '{ "userId": "analyst_01", "uniqueId": "3f7a9b2c-...", "file_name": "quarterly_planning", "question": "The stakeholders are the CTO and Product team" }'
# Generate final polished BRD (L5 — GPT-4 synthesizes everything) curl -X POST http://localhost:8501/brdfrdgeneration/final \ -H "Content-Type: application/json" \ -d '{ "userId": "analyst_01", "uniqueId": "3f7a9b2c-1d4e-..." }' # Download as Microsoft Word .docx (streamed response) curl -X POST http://localhost:8501/brdfrdgeneration/downloadbrd \ -F "userId=analyst_01" \ -F "uniqueId=3f7a9b2c-1d4e-..." \ -F "filename=Q3_Project_BRD" \ -F "htmlcontent=<html>...</html>" \ --output Q3_Project_BRD.docx # Clean up session when done curl -X POST http://localhost:8501/brdfrdgeneration/delete \ -F "userId=analyst_01" \ -F "uniqueId=3f7a9b2c-1d4e-..."
